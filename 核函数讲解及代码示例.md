
在 CUDA 编程中，**核函数（Kernel Function）** 是最核心的概念之一。简单来说，**核函数就是一段由 CPU（主机端）调用，但真正在 GPU（设备端）上并行执行的代码。**

为了理解核函数，我们可以打个比方：
假设你需要给 10 万张图片加上水印。
*   **CPU（Host）** 就像是一个总指挥，负责把图片从硬盘搬到内存，再搬到显存里。
*   **GPU（Device）** 就像是有成千上万个工人的超级工厂。
*   **核函数（Kernel）** 就像是总指挥发给每一个工人的**“标准操作手册”**。所有的工人都拿着同一本手册同时开始工作，但每个人根据自己的编号，处理不同的那一张图片。这种模式被称为 SIMT（单指令多线程）。

### 核函数的关键特征

1.  **标识符定义**：在 C/C++ 代码中，核函数必须使用 `__global__` 修饰符来声明。
2.  **执行位置**：在 GPU 上执行。
3.  **调用位置**：通常只能由 CPU 调用（较新的 CUDA 版本支持动态并行，允许 GPU 调用 GPU，但最基本的情况是 CPU 调用）。
4.  **返回值**：核函数的返回类型必须是 `void`。GPU 计算的结果需要通过传递指针的方式直接写入显存，然后再由 CPU 读回。
5.  **内置变量**：在核函数内部，可以使用 CUDA 提供的内置变量（如 `threadIdx`, `blockIdx`, `blockDim`, `gridDim`）来获取当前线程的“坐标”或“编号”，从而决定当前线程去处理哪部分数据。

### 完整代码示例：用核函数实现向量相加

为了让你直观地看到核函数长什么样以及如何使用，这里提供一个完整的、最经典的 CUDA "Hello World" 程序——**两个数组（向量）相加**。

这段代码展示了：定义核函数 -> CPU 准备数据并拷贝到 GPU -> CPU 启动核函数（千军万马开始并行计算）-> CPU 从 GPU 把结果拿回来。

```cpp
#include <iostream>
#include <cuda_runtime.h>

// ==========================================
// 1. 定义核函数 (Kernel)
// __global__ 表示这是一个核函数，在GPU上运行，由CPU调用
// ==========================================
__global__ void vectorAdd(const float *A, const float *B, float *C, int numElements) {
    // 获取当前线程的全局唯一索引 ID
    // blockDim.x: 每个线程块里面的线程数量
    // blockIdx.x: 当前线程块的编号
    // threadIdx.x: 当前线程在所在线程块里面的编号
    int i = blockDim.x * blockIdx.x + threadIdx.x;

    // 确保线程的索引没有越界
    if (i < numElements) {
        C[i] = A[i] + B[i]; // 核心操作：每个线程只做一次加法，加它负责的那一个元素
    }
}

// ==========================================
// 2. 主函数 (运行在 CPU 上)
// ==========================================
int main() {
    int numElements = 50000; // 假设向量包含5万个元素
    size_t size = numElements * sizeof(float);

    // 在 Host (CPU端) 分配内存
    float *h_A = (float *)malloc(size);
    float *h_B = (float *)malloc(size);
    float *h_C = (float *)malloc(size);

    // 初始化 Host 端的数据
    for (int i = 0; i < numElements; ++i) {
        h_A[i] = 1.0f;
        h_B[i] = 2.0f;
    }

    // 在 Device (GPU端) 分配显存
    float *d_A = nullptr;
    float *d_B = nullptr;
    float *d_C = nullptr;
    cudaMalloc((void **)&d_A, size);
    cudaMalloc((void **)&d_B, size);
    cudaMalloc((void **)&d_C, size);

    // 将数据从 Host 拷贝到 Device
    cudaMemcpy(d_A, h_A, size, cudaMemcpyHostToDevice);
    cudaMemcpy(d_B, h_B, size, cudaMemcpyHostToDevice);

    // ==========================================
    // 3. 调用核函数
    // ==========================================
    // 定义线程配置：每个Block包含256个线程
    int threadsPerBlock = 256;
    // 计算需要多少个Block才能处理完所有的元素 (向上取整)
    int blocksPerGrid = (numElements + threadsPerBlock - 1) / threadsPerBlock;

    std::cout << "启动核函数，配置：Blocks=" << blocksPerGrid << ", Threads/Block=" << threadsPerBlock << std::endl;
    
    // <<< >>> 是CUDA特有的语法，用于配置并启动核函数
    vectorAdd<<<blocksPerGrid, threadsPerBlock>>>(d_A, d_B, d_C, numElements);

    // 等待GPU上的核函数执行完毕 (通常不是必须的，因为后面的cudaMemcpy会隐式等待，这里为了演示逻辑)
    cudaDeviceSynchronize();

    // ==========================================
    // 4. 将计算结果从 Device 拷贝回 Host
    // ==========================================
    cudaMemcpy(h_C, d_C, size, cudaMemcpyDeviceToHost);

    // 验证结果
    bool success = true;
    for (int i = 0; i < numElements; ++i) {
        if (h_C[i] != 3.0f) { // 1.0 + 2.0 应该等于 3.0
            success = false;
            break;
        }
    }
    
    if (success) {
        std::cout << "计算成功！GPU 向量相加结果正确。" << std::endl;
    } else {
        std::cout << "计算出错！" << std::endl;
    }

    // 释放资源
    cudaFree(d_A);
    cudaFree(d_B);
    cudaFree(d_C);
    free(h_A);
    free(h_B);
    free(h_C);

    return 0;
}
```

**如何理解调用核函数的 `<<< >>>` 语法？**
在代码 `vectorAdd<<<blocksPerGrid, threadsPerBlock>>>(...)` 中，尖括号是配置 GPU 线程阵列的。这相当于告诉 GPU：
“请帮我执行 `vectorAdd` 这个任务，我要分发给 `blocksPerGrid` 组工人，每组包含 `threadsPerBlock` 个工人并发执行。”
